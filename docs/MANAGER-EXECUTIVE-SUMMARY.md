# RunnerSheet Trial - Manager Executive Summary

## Purpose

RunnerSheet is a browser-based digital runner sheet designed to improve journey tracking, visibility, and reporting for branch operations.

It is currently in trial and is not a replacement for the AC Runner Sheet folder until approved by branch management.

## What This Solves

- Reduces missed or unclear journey entries.
- Gives managers a single view of journey activity and corrections.
- Improves speed of checks, follow-up, and report preparation.
- Supports driver mobile usage and manager desktop usage in one system.
- Eliminates bulky paper folder handling for day-to-day journey records.
- Saves paper by moving routine tracking and reporting to digital workflows.
- Lets managers cross-check from the backend whether drivers are actively starting journeys and using the app.

## Scope for Trial

- Manager access on workplace PCs.
- Driver usage on personal mobile devices.
- Controlled branch rollout with review checkpoints.

## Manager Capabilities

- Review journeys and statuses.
- See correction context and audit trail.
- Use search/filter tools for faster investigation.
- Generate period/branch reports.
- Manage access workflows where enabled.

## Security and Access

- Role-based access (approved manager profile required).
- HTTPS enforced for site access.
- Trial use remains independently operated and manager-approved.

## IT Allowlist (Manager PCs)

Required:

- `runnersheet.win`
- `www.runnersheet.win`
- `*.googleapis.com`
- `*.gstatic.com`
- `accounts.google.com`
- `api.deepseek.com` (if manager AI search is used)

Optional:

- `api.postcodes.io` (postcode lookup workflows)

## Success Criteria for Trial

- Managers can sign in reliably from branch PCs.
- Journey review time is reduced.
- Fewer unresolved journey/correction issues.
- Reports can be generated consistently for operational checks.

## Recommended Next Step

Approve a short pilot period (2-4 weeks) at branch level, then review outcomes and decide on formal adoption.
