# RunnerSheet Trial - Manager User Guide

## 1. Before You Start

- Use a supported desktop browser on your work PC.
- Ensure your manager account is approved.
- Ensure IT allowlist is in place for required domains.

## 2. Sign In

1. Open `https://runnersheet.win`.
2. Choose **Manager sign in**.
3. Sign in with your approved account.

After sign-in, review:

- Active/recent journeys
- Review and approval areas
- Report tools
- Search/filter controls

## 4. Review Journeys

1. Go to the manager journeys page.
2. Use filters or search to find a journey.
3. Open journey details.
4. Check status, timestamps, branch, and any correction data.

Screenshot placeholder: [Journey list and filters]

## 5. Check Corrections and Audit Context

When a journey is corrected:

- Confirm reason and note.
- Check updated start/end times.
- Verify edit history and whether review is needed.

## 6. Approve or Follow Up

Where your flow supports it:

1. Approve valid records.
2. Reject/escalate records that need clarification.
3. Follow up with driver/branch where required.

Screenshot placeholder: [Approvals/review action]

## 7. Generate Reports

1. Open reports.
2. Select branch, date range, and filters.
3. Generate and review output.
4. Export/share per branch process.

Screenshot placeholder: [Report generation view]

## 8. Typical Daily Manager Routine (5-10 minutes)

1. Sign in.
2. Check flagged or corrected journeys.
3. Resolve approvals/reviews.
4. Run quick report if needed.
5. Log any issues for trial feedback.

## 9. Troubleshooting

### Cannot sign in

- Confirm manager account is approved.
- Confirm IT allowlist includes required domains.
- Try hard refresh and sign in again.

### Data not loading

- Check network and browser.
- Retry after refresh.
- If persistent, report issue with time and screenshot.

### AI search unavailable

- Confirm `api.deepseek.com` is allowed by IT.
- Continue with standard filters/search until restored.
